# Dependencies.toml Reference

The `Dependencies.toml` file is an auto-generated lock file that records the exact versions of all direct and transitive dependencies resolved for a Ballerina package. This file ensures reproducible builds across environments and should be committed to version control.

## Overview

| Aspect | Detail |
|--------|--------|
| File location | Project root directory, alongside `Ballerina.toml` |
| Generated by | `bal build`, `bal run`, `bal test`, or `bal pack` |
| Purpose | Lock resolved dependency versions for reproducible builds |
| Manual editing | Not recommended; regenerate with `bal build` |
| Version control | Should be committed to the repository |

## File Structure

The file uses the [TOML](https://toml.io) format and contains the following sections.

### `[ballerina]`

Records the Ballerina distribution version used to resolve the dependencies.

```toml
[ballerina]
dependencies-toml-version = "2"
distribution-version = "2201.9.0"
```

| Key | Type | Description |
|-----|------|-------------|
| `dependencies-toml-version` | string | Format version of the Dependencies.toml file (currently `"2"`) |
| `distribution-version` | string | Ballerina distribution version used for resolution |

### `[[package]]`

Each resolved package (direct or transitive) is listed as a `[[package]]` entry.

```toml
[[package]]
org = "ballerina"
name = "http"
version = "2.11.2"
scope = "default"
transitive = false

[[package.dependency]]
org = "ballerina"
name = "io"

[[package.dependency]]
org = "ballerina"
name = "mime"

[[package]]
org = "ballerina"
name = "io"
version = "1.6.1"
scope = "default"
transitive = true
```

### Package Entry Fields

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `org` | string | Yes | Organization name of the package |
| `name` | string | Yes | Package name |
| `version` | string | Yes | Resolved version |
| `scope` | string | No | Dependency scope: `"default"` or `"testOnly"` |
| `transitive` | boolean | No | `true` if the package is a transitive (indirect) dependency |

### `[[package.dependency]]`

Lists the direct dependencies of each package.

| Key | Type | Description |
|-----|------|-------------|
| `org` | string | Organization name of the dependency |
| `name` | string | Package name of the dependency |

### `[[package.module]]`

Lists the modules within each package that are used by the project.

```toml
[[package]]
org = "ballerina"
name = "http"
version = "2.11.2"

[[package.module]]
org = "ballerina"
packageName = "http"
moduleName = "http"

[[package.module]]
org = "ballerina"
packageName = "http"
moduleName = "http.httpscerr"
```

| Key | Type | Description |
|-----|------|-------------|
| `org` | string | Organization name |
| `packageName` | string | Parent package name |
| `moduleName` | string | Fully qualified module name |

## Version Resolution

Ballerina uses the following rules to resolve dependency versions.

| Rule | Description |
|------|-------------|
| Latest compatible | Resolves to the latest version that satisfies the version constraint in `Ballerina.toml` |
| Minimum version selection | When multiple packages depend on different versions, the highest minimum version is selected |
| Distribution compatibility | Only packages compatible with the current distribution version are considered |
| Lock file priority | If `Dependencies.toml` exists, locked versions are used unless a newer compatible version is required |
| Sticky versions | Running `bal build` preserves existing locked versions; use `bal build --sticky=false` to force re-resolution |

## Repository Configuration

Dependencies are resolved from configured repositories in the following order.

| Priority | Repository | Description |
|----------|-----------|-------------|
| 1 | Local repository | `~/.ballerina/repositories/local` |
| 2 | Custom repositories | Configured in `<ballerina-home>/Settings.toml` |
| 3 | Ballerina Central | `https://central.ballerina.io` (default remote repository) |

### Settings.toml Repository Configuration

```toml
# ~/.ballerina/Settings.toml
[central]
accesstoken = "<your-access-token>"

[[repository.custom]]
id = "wso2-nexus"
url = "https://maven.wso2.org/nexus/content/repositories/releases/"
username = "user"
password = "pass"
```

## Regenerating Dependencies.toml

| Command | Behavior |
|---------|----------|
| `bal build` | Resolves and locks dependencies; preserves existing versions (sticky) |
| `bal build --sticky=false` | Re-resolves all dependencies to latest compatible versions |
| Delete `Dependencies.toml` and run `bal build` | Full re-resolution from scratch |
